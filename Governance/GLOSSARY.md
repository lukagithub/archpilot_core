# ArchPilot Core 术语标准表

**文档版本**: v1.0.0  
**最后更新**: 2026-02-01  
**适用范围**: 基于 ArchPilot Core 框架的所有项目  
**引用自**: `Governance/GOVERNANCE_OVERVIEW.md`

> 所有治理文档、脚本输出、模板与审计报告必须使用本术语表中的标准写法。若需新增术语，必须先更新此文件。

---

## 1. 核心术语

| 中文术语 | 英文 | 缩写 | 定义 | 规范写法 |
|----------|------|------|------|----------|
| L1需求层 | L1 Requirements Layer | L1 | 功能需求、验收条件、业务指标 | 文档中统一写作"L1需求层" |
| L2架构层 | L2 Architecture Layer | L2 | 系统抽象、模块关系、接口划分 | 统一写作"L2架构层" |
| L3设计层 | L3 Detail Design Layer | L3 | 详细设计、接口契约、算法说明 | 统一写作"L3设计层" |
| L4实现层 | L4 Implementation Layer | L4 | 源代码、配置、运行时 artefact | 统一写作"L4实现层" |
| L5验证层 | L5 Verification Layer | L5 | 测试/验证 artefact 与报告 | 统一写作"L5验证层" |
| 追溯关系 | Traceability | - | L1→L5 闭环的关联关系 | 中文优先使用"追溯关系" |
| 加权总分 | Weighted Score | - | 五维度评分的最终得分 | 与质量评分体系一致 |
| 质量评级 | Quality Grade | - | 优秀/良好/及格/不及格四档 | 禁止使用"合格/不合格" |
| 发布版本 | Release Version | - | 语义化版本号，如 v1.2.3 | 统一写作"发布版本" |
| 治理规则 | Governance Rule | - | 约束 AI 和人工行为的标准 | 统一写作"治理规则" |

## 2. 规则等级（RFC 2119）

| 图标 | 术语 | 英文 | 含义 | 示例 |
|------|------|------|------|------|
| ✅ | MUST | MUST | 绝对强制要求，违反即视为阻断项 | "文档 MUST 包含元数据" |
| ⚠️ | SHOULD | SHOULD | 推荐实施，如有合理理由可豁免 | "测试 SHOULD 覆盖边界条件" |
| ⭕ | MAY | MAY | 可选项，无强制性 | "MAY 附加性能基线数据" |
| ❌ | MUST NOT | MUST NOT | 禁止执行 | "MUST NOT 硬编码配置路径" |

> 文档中可同时使用 emoji + RFC 2119 术语，如"✅ MUST 遵循命名规范"

## 3. 质量评级术语

| 评级 | 分数范围 | 英文 | 决策建议 |
|------|----------|------|----------|
| 优秀 | ≥4.5 | Excellent | 推荐正式发布 |
| 良好 | ≥4.0 且 <4.5 | Good | 可以正式发布 |
| 及格 | ≥3.0 且 <4.0 | Pass | 仅限 Beta/RC |
| 不及格 | <3.0 | Fail | 禁止发布 |

**注意**: 该分段为默认配置，项目可在 `rules/rules_release.md` 中调整。

## 4. 版本管理术语

| 术语 | 英文 | 含义 |
|------|------|------|
| 主版本号 | MAJOR | 不兼容的 API 变更 |
| 次版本号 | MINOR | 向后兼容的功能新增 |
| 修订号 | PATCH | 向后兼容的问题修复 |
| 预发布版 | Pre-release | alpha/beta/rc 版本 |
| 构建信息 | Build Metadata | 附加的构建标识 |

## 5. 文档状态术语

| 状态 | 英文 | 含义 | 后续操作 |
|------|------|------|----------|
| 草稿 | draft | 初稿，未经评审 | 提交评审 |
| 评审中 | review | 评审进行中 | 等待批准 |
| 已批准 | approved | 正式生效 | 可被引用 |
| 已废弃 | deprecated | 不再使用 | 禁止引用 |

## 6. 追溯关系术语

| 术语 | 字段名 | 含义 |
|------|--------|------|
| 来源追溯 | traces_from | 当前文档派生自哪些上游文档 |
| 目标追溯 | traces_to | 当前文档被哪些下游文档实现 |
| 关联文档 | related | 相关但非直接追溯的文档 |
| 验证关系 | verified_by | 被哪些测试用例验证 |

## 8. 使用指南

### 8.1 文档作者

- 撰写任何文档时使用本表术语
- 若发现未定义术语，先更新本表再继续撰写

### 8.2 脚本输出

- 所有 CLI/报告脚本须引用本表中的写法
- 避免自由文本如 "PASS/FAIL"，使用标准术语

### 8.3 AI 执行

- AI 在生成内容时必须与本表一致
- 若出现冲突，以本表为准

### 8.4 项目定制

- 项目特定术语可在本表末尾添加
- 添加时需注明适用范围

## 10. 变更流程

1. **提议**: 在 Issue 中登记术语变更需求
2. **评审**: 治理委员会评审术语是否会影响既有文档/脚本
3. **生效**: 合并后本表立即生效，应同步更新相关文档
